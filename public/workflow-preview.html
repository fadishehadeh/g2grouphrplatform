Before

Typical pain points

  • Balance is visible only after submission or on another page.
  • Users discover blocked dates too late.
  • Approver path is unclear, so staff ask HR manually.
  • One long form mixes inputs, warnings, and policy text together.

After

Workflow upgrades

  • Step-by-step progression with visible completion state.
  • Immediate policy feedback beside the form, not after submit.
  • Sticky summary panel so the user always sees impact.
  • Draft/save pattern for interrupted work without changing logic.

Same Backend, Better Flow

What changes in the real app

Keep

Routes, permissions, leave calculations, approval rules, and database structure.

Improve

Form grouping, step order, summary visibility, inline validation, and feedback states.

Result

Faster completion, fewer user mistakes, and less HR support overhead.
